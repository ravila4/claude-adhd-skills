### Obsidian Semantic Search - Setup and Enhancement

**10:00 AM** - Started work on obsidian-semantic.

Cloned repo, installed with `uv sync`. Tool is a CLI that indexes Obsidian vault markdown into embeddings and enables semantic search and related-note discovery.

#### Embedding Model Research

Evaluated two Ollama embedding models for local use on Mac M1:

| | nomic-embed-text | qwen3-embedding:8b |
|---|---|---|
| **Warm-up** | 1.7s | 15.7s |
| **Short note** (~73 chars) | ~0.07s | ~0.27s |
| **Medium note** (~617 chars) | ~0.07s | ~0.46s |
| **Long note** (~1664 chars) | ~0.06s | ~0.78s |
| **Batch throughput** (10 items) | 0.02s/item | 0.36s/item |
| **Dimensions** | 768 (fixed) | 32-4096 (configurable) |
| **Model size** | 274 MB | 4.7 GB |

**Decision:** Went with qwen3-embedding:8b at 4096 dimensions. Sub-second latency on M1 is acceptable for a personal vault.

#### Instruction Prefix Feature

Discovered that qwen3-embedding supports instruction-aware embeddings: queries use `Instruct: ...\nQuery: ` prefix, documents go as raw text.

**Problem:** The tool's Ollama embedder treated all models identically, with no document/query distinction.

**Implemented:** Config-driven prefix system (TDD, 130 tests passing):

- Added `embed_query()` and `embed_document()` to base `Embedder` class
- Ollama embedder: prepends configurable `query_prefix` / `document_prefix` strings
- Config: optional `query_prefix` and `document_prefix` in `embedder:` section of YAML

Full vault indexed: **129 files, 540 chunks** in a single pass.
